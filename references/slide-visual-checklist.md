# Slide Visual Checklist

「問題がある前提で検査する」を基本姿勢とする。

## 基本原則: スライドの視覚的一貫性

スライドを検査する際、個々のスライドだけでなくデッキ全体としての一貫性を考える。
同種の要素は統一されたスタイルで表示され、情報の視覚的な階層が保たれていることを確認する。

例:

- セクションタイトルスライド → 同じフォントサイズ、同じレイアウト
- 箇条書きスライド → 同じインデント、同じ行間
- 図解スライド → 同じ余白、同じキャプション位置

この原則は個別チェック項目より上位に位置する。チェックリストに明示されていない一貫性の問題でも、論理的に導けるものは検査対象とする。

**signal-to-noise（運用ヒューリスティック）**: 装飾テンプレート・全ページのロゴ・ドロップシャドウ・過剰な区切り線などは「ノイズ」。情報伝達に効かない装飾が信号（中身）を埋もれさせていないかを併せて見る。「取り除けるものは取り除く」（Tufte の data-ink、Reynolds *Presentation Zen* の signal-to-noise）。※一貫性軸に減算原則を混ぜる扱いなので、ハードな判定でなく横断原則として用いる。出典: Tufte 1983 *The Visual Display of Quantitative Information* ／ Reynolds *Presentation Zen*。

## チェックリスト（10観点）

### 1. テキスト折り返し

- [ ] タイトルがスライド幅に収まっている（不自然な改行がない）
- [ ] 箇条書きの各項目が意味の切れ目で折り返されている
- [ ] コードブロックが横方向にはみ出していない
- [ ] URLや長い英単語が不自然に折り返されていない
- [ ] テーブルのセル内テキストが潰れていない
- [ ] CJK禁則: 行頭に句読点・閉じ括弧、行末に開き括弧が来ていない。英単語/数式/コードが途中で不自然に分断されていない（運用ヒューリスティック・体系的な一次ソースなし）
- [ ] 「→」など記号の前後で、汚い自動折り返しが起きていないか。長い結論文ほど起きやすい → 短文化、または「→」を行頭にする手動改行で整える（CJKは `word-break: keep-all` / 手動 `<br>` で対処）
- [ ] スライド文末に**不要な「。」**が付いていないか（文・節を分ける区切りの「。」は可。結論行末の飾りの「。」は外す）

### 2. 要素の被り

- [ ] テキスト同士が重なっていない
- [ ] テキストと画像/図解が重なっていない
- [ ] 画像同士が重なっていない
- [ ] ヘッダー/フッター領域とコンテンツ領域が干渉していない
- [ ] ページ番号やロゴが他の要素を隠していない

### 3. 余白バランス

- [ ] 上下左右の余白が均等である（極端な偏りがない）
- [ ] コンテンツが中央寄せまたは意図的な配置になっている
- [ ] **結論（赤文字Punch等）や補足が、画面下部に行きすぎていないか**。本文との間隔が空きすぎなら、本文の直下に来るよう上に詰める（**タイトル位置は固定。下側に余白が残るのは問題ない＝空きを埋めにいかない**）
- [ ] 要素間の間隔が適切で、詰まりすぎ/空きすぎがない
- [ ] コンテンツがスライド領域の50%以上を活用している

### 4. フォントサイズ

- [ ] 本文テキストが判読可能なサイズである（プロジェクション想定で24pt以上推奨）
- [ ] タイトルと本文のサイズに明確な差がある（視覚的階層）
- [ ] 注釈・キャプションが小さすぎない（16pt以上推奨）
- [ ] コードブロックのフォントサイズが本文と同等以上
- [ ] 複数スライドでフォントサイズが統一されている

> **PNGからptは測れない → 2段階で確認**: (1) PNGの文字の高さ（スライド高に対する比率）で「小さすぎ候補」を当たりをつけ、(2) Marpテーマの CSS / PPTX のフォント指定を読んで正確なptを確認する。比率の目安（例: スライド高の約2.2%未満＝24pt未満が疑わしい）は**運用ヒューリスティック**で一次ソースなし。24pt/16ptの数値は投影時の業界的な経験則（学術的一次ソースなし・「推奨」のまま）。

### 5. 画像品質

- [ ] 画像の解像度が十分で、ピクセルが粗くない
- [ ] アスペクト比が歪んでいない（横伸び/縦伸びがない）
- [ ] 画像の背景色がスライドの背景と調和している
- [ ] スクリーンショットのテキストが判読可能
- [ ] 画像のサイズがスライド内で適切（大きすぎ/小さすぎがない）
- [ ] 隅（右上等）に小さく置く画像には、**何の画像か分かるキャプションを付ける**（無いと浮く）
- [ ] 画像を大きく見せたいときは、**半透明（opacity 0.2〜0.3）で背景的に敷く**手もある（前面テキストのコントラストは確保する）

### 6. レイアウト一貫性

- [ ] 同じ種類のスライド（タイトル、本文、図解）が統一されたレイアウトを使っている
- [ ] テキストの開始位置が連続スライドで揃っている
- [ ] 画像の配置パターンが統一されている（左寄せ/中央/右寄せ）
- [ ] 色使いがデッキ全体で一貫している
- [ ] 見出しレベルごとのスタイルが統一されている

### 7. コントラスト（輝度）

WCAG 2.2 SC 1.4.3。**四捨五入しない**（4.49:1 は不合格）。

- [ ] 本文テキスト ≥ 4.5:1
- [ ] 大きい文字（≥18pt、または太字 ≥14pt）≥ 3:1
- [ ] 意味を持つ図形要素・チャートの線/棒・境界線 ≥ 3:1（WCAG 2.2 SC 1.4.11 非テキストコントラスト＝1.4.3とは別基準）
- [ ] 薄い色のテキストが背景に溶け込んでいない／暗い背景上のテキストが十分明るい

> **PNGでの測り方**: 際どい箇所の前景色・背景色のRGBをPNG上で推定してコントラスト比を出す。確証が要れば元のCSS/テーマの色指定を読む。出典: W3C WCAG 2.2 SC 1.4.3 https://www.w3.org/WAI/WCAG22/Understanding/contrast-minimum.html ／ SC 1.4.11 https://www.w3.org/WAI/WCAG21/Understanding/non-text-contrast.html 。

### 8. 色のみに依存していないか（WCAG 1.4.1）

SC 1.4.1（Use of Color）は 1.4.3（輝度コントラスト）とは**独立した別基準**。色だけで区別している箇所はグレースケール/色覚特性で破綻する。

- [ ] 系列を色だけで区別していない（直接ラベル・パターン・記号も併用）
- [ ] 赤/緑だけの凡例になっていない
- [ ] 「赤い項目を見て」など色だけを参照する指示がない
- [ ] 強調を色だけに頼っていない（太字・下線・位置も）

出典: W3C WCAG 2.2 SC 1.4.1 https://www.w3.org/WAI/WCAG22/Understanding/use-of-color.html ／ Knaflic「色は控えめに・直接ラベル」。

### 9. チャートの誠実さ（Tufte）

スライド上のグラフがミスリードしていないか（PNGで見える範囲で）。

- [ ] 棒グラフの基線が0から始まっている（途中切りで差を誇張していない）
- [ ] 軸ラベル・単位がある
- [ ] 1次元の量を面積/体積で表していない（誇張）。3D円グラフ等で歪んでいない
- [ ] データに対して視覚変化が比例している（Lie Factor ≒ 1）／チャートジャンク（moiré・過剰装飾）がない

出典: Tufte 1983 *The Visual Display of Quantitative Information*（Lie Factor / data-ink / graphical integrity）。※これは**定量グラフ**の話。写真の縦横比の歪み（観点5）はこの軸ではなく素のレイアウト判断。

### 10. グランステスト（1スライド1メッセージ・視覚の焦点）

~3秒で「どこを見ればいいか」が掴めるか。**視覚の焦点・密度のみを見る**（メッセージ内容の良し悪しは review-content の担当）。

- [ ] 1スライドに視覚的な焦点が1つ（要素を詰め込みすぎていない）
- [ ] 視線誘導（サイズ・位置・コントラスト）で主役が即わかる

出典: Duarte slide:ology「Glance Test」 https://hbr.org/2012/10/do-your-slides-pass-the-glance-test ／ Reynolds signal-to-noise。

## Marp固有の追加観点

- [ ] テーマのCSS変数がスライド全体に正しく適用されている
- [ ] `<!-- _class: xxx -->` によるレイアウト切り替えが意図通り機能している
- [ ] 背景画像（`![bg](...)`) のサイズと位置が適切
- [ ] 分割レイアウト（`![bg left](...)` 等）でテキストと画像のバランスが取れている
- [ ] Mermaid/数式の表示が崩れていない

## PPTX固有の追加観点

- [ ] マスターレイアウトのプレースホルダーが正しく使われている
- [ ] フォントがフォールバックされていない（意図したフォントで表示されている）
- [ ] アニメーション設定がある場合、静止状態で全要素が視認可能
