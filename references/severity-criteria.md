# Severity Criteria

slide-visual-reviewerの問題分類基準。

## 3段階の重要度

### error（修正必須）

情報の欠落や重大な視認性問題を引き起こす問題。修正せずに提出・発表してはならない。

|判定条件|例|
|-|-|
|テキストが判読不能|フォントサイズが小さすぎる、コントラスト不足で見えない|
|要素の被りで情報が隠れる|テキストが画像に完全に隠されている|
|コンテンツのはみ出し|テキストやコードがスライド領域外に切れている|
|画像のアスペクト比が明らかに歪んでいる|人物や製品の画像が横伸び/縦伸びしている|
|スライド間で重大な不整合|同じ種類のスライドでフォントサイズが倍以上異なる|

### warning（修正推奨）

視認性や美観に影響するが、情報自体は伝わる問題。可能な限り修正する。

|判定条件|例|
|-|-|
|テキストの不自然な折り返し|タイトルが2行に折り返されるが読める|
|余白の軽微な偏り|片側の余白が反対側の1.5倍程度|
|フォントサイズの軽微な不統一|同種スライドでフォントサイズが2pt程度異なる|
|画像の解像度がやや不足|若干のピクセル化があるが内容は判読可能|
|要素間の間隔が不均一|要素はすべて見えるが、間隔にばらつきがある|

### info（参考情報）

品質向上の余地があるが、発表に支障はない問題。時間に余裕がある場合に対応する。

|判定条件|例|
|-|-|
|余白の微細な調整余地|5px程度の余白差|
|配色の最適化余地|コントラストは十分だが、より良い色選択が可能|
|レイアウトの改善提案|現状でも問題ないが、より洗練された配置が可能|
|アクセシビリティ向上の余地|高コントラストモードへの対応|

## 判定フローチャート

```
問題を検出
  ├→ 情報が欠落・判読不能か？ → YES → error
  ├→ 視認性・美観に明確な影響があるか？ → YES → warning
  └→ 改善の余地はあるが機能的には問題ないか？ → YES → info
```

## 総合判定（リリースゲート）

個々の severity は数値集計しない。代わりに**最悪値ゲート**（worst-of）で発表可否を決める:

- **error が1つでもある → FAIL（発表不可）**
- **error = 0、warning がある → PASS WITH FIXES（直してから）**
- **info のみ → PASS**

> **なぜ最悪値ゲートか（根拠）**: これは「研究で視覚欠陥は補償不能だから」ではない（その主張に出典はない）。**severity の定義そのものの論理的帰結**である。error は「情報の欠落・判読不能・隠れて見えない」を意味する以上、別スライドが優れていても相殺できない。だから review-content の最頻値（MODE）のような平均的集計は持ち込まず、worst-of ゲートを使う。

## 程度で判定する（単一の数値プロキシに固定しない）

severity は**影響の程度の連続体**で判定する。表中の「倍以上」「1.5倍」「2pt」「5px」は目安の例であって閾値ではない。例（コントラスト）:

- **error**: AAの 4.5:1 を満たさず実際に読みづらい
- **warning**: 4.5:1 は満たすが余裕が薄い（細字・小サイズで不安）
- **info**: AAA 7:1 には届かない

## 根拠アンカー（必須）

各指摘は、PNG上の場所を特定して引用する: **スライド番号 ＋ 要素 ＋ 観察した状態**（例:「スライド3・本文箇条書き最終項目・フッター画像と重なり判読不能」）。これによりスライド間のハロー効果（良い印象/悪い印象の波及）を抑える。

判定方法論の出典: Brookhart（連続的アンカー記述）。

## レポートでの使い方

問題をリストアップする際、各問題に severity を付与する:

```
- [error] スライド3: 箇条書きテキストがフッター画像と重なり、最後の項目が読めない
- [warning] スライド5: タイトルが2行に折り返されている（「〜について」で改行）
- [info] スライド8: 左余白が右余白よりやや広い
```
