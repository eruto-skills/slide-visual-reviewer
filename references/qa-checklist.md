# QA Checklist

共通チェック項目は [qa-checklist-base](../_shared/qa-checklist-base.md) を参照。以下はドメイン固有の項目。

スライドはプロジェクション環境で視聴されるため、画面上では気にならない問題も投影時には致命的になる。

## Failure Modes（頻出問題）

- **テキスト折り返し**: タイトルや箇条書きが不自然な位置で改行され、読みにくい
- **要素の被り**: テキストが画像やフッターと重なり、情報が失われる
- **余白の偏り**: コンテンツが片側に寄り、プロフェッショナルさを損なう
- **フォントサイズ不足**: 後方座席から判読できないサイズのテキスト
- **一貫性の欠如**: 同種のスライドでレイアウトやフォントサイズが異なる
- **コントラスト不足**: 薄い色のテキストがプロジェクターで見えなくなる
- **画像品質問題**: 低解像度画像がスライド上で粗く表示される

## Checklist

### A. PNG生成確認

- [ ] 全スライドのPNGが正常に生成された
- [ ] PNGの解像度がレビューに十分（幅1920px以上推奨）
- [ ] 生成されたPNG枚数がスライド枚数と一致する

### B. 個別スライド検査

→ [slide-visual-checklist](slide-visual-checklist.md) の7観点で検証

- [ ] テキスト折り返し（不自然な改行がない）
- [ ] 要素の被り（テキスト・画像・UI要素の重なりがない）
- [ ] 余白バランス（均等で適切な余白）
- [ ] フォントサイズ（判読可能なサイズ）
- [ ] 画像品質（解像度十分、アスペクト比正常）
- [ ] レイアウト一貫性（デッキ全体で統一）
- [ ] コントラスト・視認性（十分なコントラスト）

### C. デッキ全体の検査

- [ ] スライド間のレイアウト遷移が自然
- [ ] 色使い・フォントがデッキ全体で統一されている
- [ ] 情報密度が均一（極端に詰まったスライドと空のスライドが混在しない）
- [ ] セクション区切りが視覚的に明瞭

### D. 対象形式固有の検査（該当する方）

**Marp**:

- [ ] テーマCSSの適用が全スライドで正しい
- [ ] 分割レイアウトのバランスが適切
- [ ] 背景画像の表示が意図通り

**PPTX**:

- [ ] マスターレイアウトが正しく適用されている
- [ ] フォントフォールバックが発生していない
- [ ] プレースホルダーの位置が適切

## Verification Loop

1. PNG生成（marp-validate.js --png / soffice --convert-to png）
2. 各PNGをRead toolで視覚確認
3. 問題を列挙（severity付き）
4. 修正案を提示
5. ユーザー承認後に修正
6. 再検証（Step 1-2を再実行）
7. 指摘の抽象化: 具体的な問題を汎用的な検査観点に昇華し、[slide-visual-checklist](slide-visual-checklist.md) に追記

> [!NOTE]
> 汎用的な5ステップVerification Loopはベースを参照。上記はビジュアルレビュー固有の拡張ループ。
